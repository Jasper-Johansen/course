# ആമുഖം[[introduction]]

ഹഗ്ഗിങ് ഫെയ്സ് (Hugging Face) കോഴ്സിലേക്ക് സ്വാഗതം! നിങ്ങളുടെ വർക്കിങ് എൻവയോൺമെന്റ് (working environment) എങ്ങനെ സജ്ജീകരിക്കാം എന്നാണ് ഈ ആമുഖത്തിലൂടെ പരിചയപ്പെടുത്തുന്നത്. നിങ്ങൾ ഈ കോഴ്‌സ് ആരംഭിക്കുന്നതേയുള്ളൂ എങ്കിൽ, ആദ്യം [അധ്യായം 1](/course/chapter1) നോക്കി വരാൻ ഞങ്ങൾ നിർദ്ദേശിക്കുന്നു. അതിനുശേഷം ഇവിടെ തിരിച്ചെത്തി എൻവയോൺമെന്റ് സജ്ജീകരിക്കുന്നതിലൂടെ കോഡുകൾ നിങ്ങൾക്ക് സ്വയം പരീക്ഷിച്ചുനോക്കാൻ സാധിക്കുന്നതാണ്.

ഈ കോഴ്സിൽ നമ്മൾ ഉപയോഗിക്കുന്ന ലൈബ്രറികളെല്ലാം (libraries) പൈത്തൺ പാക്കേജുകളായി (python packages) ലഭ്യമാണ്. അതുകൊണ്ടുതന്നെ, ഒരു പൈത്തൺ എൻവയോൺമെന്റ് എങ്ങനെ സജ്ജീകരിക്കാമെന്നും ഈ കോഴ്സിന് ആവശ്യമായ പ്രത്യേക ലൈബ്രറികൾ എങ്ങനെ ഇൻസ്റ്റാൾ ചെയ്യാമെന്നും നമുക്കിവിടെ നോക്കാം.

നമ്മുടെ വർക്കിങ് എൻവയോൺമെന്റ് രണ്ട് രീതിയിൽ സജ്ജമാക്കാവുന്നതാണ്. ഒന്ന് ഗൂഗിൾ കൊളാബ് നോട്ട്ബുക്ക് (Google Colab Notebook) വഴിയും, മറ്റൊന്ന് പൈത്തൺ വിർച്വൽ എൻവയോൺമെന്റ് (virtual environment) വഴിയും. ഇതിൽ നിങ്ങൾക്ക് ഏറ്റവും അനുയോജ്യമെന്ന് തോന്നുന്ന രീതി തിരഞ്ഞെടുക്കാം. എങ്കിലും, തുടക്കക്കാർ കൊളാബ് നോട്ട്ബുക്ക് ഉപയോഗിക്കുന്നതാവും നല്ലത്.

ഈ കോഴ്സിൽ വിൻഡോസ് (Windows) സിസ്റ്റത്തിനായുള്ള നിർദ്ദേശങ്ങൾ ഉൾപ്പെടുത്തിയിട്ടില്ല. നിങ്ങൾ വിൻഡോസ് ആണ് ഉപയോഗിക്കുന്നതെങ്കിൽ, ഒരു കൊളാബ് നോട്ട്ബുക്ക് വഴി കോഴ്‌സ് പിന്തുടരാൻ ശ്രദ്ധിക്കുക. ഇനി നിങ്ങൾ ഏതെങ്കിലും ലിനക്സ് ഡിസ്ട്രിബ്യൂഷനുകളോ (Linux distributions) മാക് ഒഎസോ (macOS) ആണ് ഉപയോഗിക്കുന്നതെങ്കിൽ, ഇവിടെ പറഞ്ഞിരിക്കുന്ന രണ്ട് രീതികളിൽ ഏത് വേണമെങ്കിലും തിരഞ്ഞെടുക്കാവുന്നതാണ്.

ഈ കോഴ്സിലെ മിക്ക ഭാഗങ്ങളും പിന്തുടരുന്നതിന് നിങ്ങൾക്ക് ഒരു ഹഗ്ഗിങ് ഫെയ്സ് അക്കൗണ്ട് ആവശ്യമാണ്. അതുകൊണ്ട് ഇപ്പോൾത്തന്നെ ഒരു അക്കൗണ്ട് തുടങ്ങാൻ ഞങ്ങൾ നിർദ്ദേശിക്കുന്നു: [പുതിയ അക്കൗണ്ട് തുറക്കുക](https://huggingface.co/join).

## ഗൂഗിൾ കൊളാബ് വഴി[[using-a-google-colab-notebook]]

കൊളാബ് നോട്ട്ബുക്ക് ഉപയോഗിക്കുന്നതാണ് ഏറ്റവും ലളിതമായ രീതി. നിങ്ങളുടെ ബ്രൗസറിൽ നോട്ട്ബുക്ക് തുറന്നാൽ നേരിട്ട് കോഡിംഗിലേക്ക് കടക്കാവുന്നതാണ്!

നിങ്ങൾക്ക് കൊളാബ് മുൻപ് ഉപയോഗിച്ച് പരിചയമില്ലെങ്കിൽ, ആദ്യം ഈ [ആമുഖം](https://colab.research.google.com/notebooks/intro.ipynb) നോക്കി വരുന്നത് ഉചിതമായിരിക്കും. ചെറിയ ആവശ്യങ്ങൾക്കായി ജി.പി.യു (GPU) അല്ലെങ്കിൽ ടി.പി.യു (TPU) പോലുള്ള വേഗതയേറിയ ഹാർഡ്‌വെയറുകൾ സൗജന്യമായി ഉപയോഗിക്കാൻ കൊളാബ് നിങ്ങളെ അനുവദിക്കുന്നുണ്ട്.

കൊളാബ് ഉപയോഗിക്കുന്നതിൽ നിങ്ങൾക്ക് ആത്മവിശ്വാസം ലഭിച്ചുവെങ്കിൽ, ഇനി നമുക്ക് സജ്ജീകരണങ്ങളിലേക്ക് കടക്കാം. അതിനായി പുതിയൊരു നോട്ട്ബുക്ക് തുറക്കുക:

ഈ കോഴ്‌സിന് ആവശ്യമായ ലൈബ്രറികൾ ഇൻസ്റ്റാൾ ചെയ്യുന്നതാണ് അടുത്ത ഘട്ടം. പൈത്തണിന്റെ പാക്കേജ് മാനേജരായ `pip` ഉപയോഗിച്ചാണ് നമ്മൾ ഇത് ചെയ്യുന്നത്. നോട്ട്ബുക്കുകളിൽ സിസ്റ്റം കമാൻഡുകൾ പ്രവർത്തിപ്പിക്കുന്നതിനായി കമാൻഡിനു മുൻപായി `!` ചിഹ്നം ചേർക്കാറുണ്ട്. അതുകൊണ്ട്, താഴെ പറയുന്ന രീതിയിൽ നിങ്ങൾക്ക് 🤗 Transformers ലൈബ്രറി ഇൻസ്റ്റാൾ ചെയ്യാവുന്നതാണ്:

```
!pip install transformers
```
ഇനി ഈ പാക്കേജ് വിജയകരമായി ഇൻസ്റ്റാൾ ആയോ എന്നറിയാൻ നിങ്ങളുടെ പൈത്തൺ റൺടൈമിൽ (Python runtime) ഇതൊന്ന് ഇംപോർട്ട് ചെയ്തു നോക്കിയാൽ മതി:

```
import transformers
```

ഇത് 🤗 Transformers-ന്റെ വളരെ ലളിതമായ ഒരു പതിപ്പാണ് ഇൻസ്റ്റാൾ ചെയ്യുന്നത്. PyTorch-ഉം TensorFlow-ഉം പോലുള്ള മെഷീൻ ലേണിംഗ് ഫ്രെയിംവർക്കുകളൊന്നും ഇതിലൂടെ ഇൻസ്റ്റാൾ ചെയ്യപ്പെടില്ലെന്ന് ശ്രദ്ധിക്കുക. നമ്മൾ ഈ ലൈബ്രറിയുടെ വൈവിധ്യമാർന്ന നിരവധി ഫീച്ചറുകൾ ഉപയോഗിക്കുന്നതിനാൽ, ഇതിന്റെ ഡെവലപ്‌മെന്റ് വേർഷൻ (development version) ഇൻസ്റ്റാൾ ചെയ്യുന്നതാണ് ഏറ്റവും നല്ലത്. നമ്മുടെ മിക്ക ആവശ്യങ്ങൾക്കും വേണ്ട ഡിപെൻഡൻസികൾ (dependencies) ഇതിൽ ഉൾപ്പെട്ടിട്ടുണ്ട്:

```
!pip install transformers[sentencepiece]
```
ഇതിന് അൽപം സമയമെടുക്കും, എങ്കിലും ഇത് പൂർത്തിയാകുന്നതോടെ ഈ കോഴ്സിലെ ബാക്കി ഭാഗങ്ങളിലേക്ക് കടക്കാൻ നിങ്ങൾ പൂർണ്ണസജ്ജരാകും!

## പൈത്തൺ വിർച്വൽ എൻവയോൺമെന്റ് വഴി[[using-a-python-virtual-environment]]

നിങ്ങൾക്ക് ഒരു പൈത്തൺ വിർച്വൽ എൻവയോൺമെന്റ് ഉപയോഗിക്കാനാണ് താൽപര്യമെങ്കിൽ, അതിന്റെ ആദ്യ പടിയായി നിങ്ങളുടെ സിസ്റ്റത്തിൽ പൈത്തൺ ഇൻസ്റ്റാൾ ചെയ്യേണ്ടതുണ്ട്. ഇതിനായി [ഈ നിർദ്ദേശങ്ങൾ](https://realpython.com/installing-python/) പിന്തുടരുക.

പൈത്തൺ ഇൻസ്റ്റാൾ ചെയ്തുകഴിഞ്ഞാൽ, നിങ്ങളുടെ ടെർമിനലിൽ പൈത്തൺ കമാൻഡുകൾ പ്രവർത്തിപ്പിക്കാൻ നിങ്ങൾക്ക് സാധിക്കും. അടുത്ത ഘട്ടങ്ങളിലേക്ക് കടക്കുന്നതിന് മുൻപ്, പൈത്തൺ ഇൻസ്റ്റലേഷൻ കൃത്യമാണെന്ന് ഉറപ്പുവരുത്താൻ ഈ കമാൻഡ് റൺ ചെയ്യുക: `python --version`. ഇത് നിങ്ങളുടെ സിസ്റ്റത്തിൽ ഇപ്പോൾ ലഭ്യമായ പൈത്തൺ പതിപ്പ് ഏതാണെന്ന് കാണിച്ചുതരും.

`python --version` പോലുള്ള ഒരു പൈത്തൺ കമാൻഡ് ടെർമിനലിൽ റൺ ചെയ്യുമ്പോൾ, ആ കമാൻഡ് പ്രവർത്തിപ്പിക്കുന്ന പ്രോഗ്രാമിനെ നിങ്ങളുടെ സിസ്റ്റത്തിലെ അടിസ്ഥാന പൈത്തൺ ആയി കരുതാം. ഈ അടിസ്ഥാന ഇൻസ്റ്റലേഷനിൽ മറ്റ് പാക്കേജുകളൊന്നും ചേർക്കാതെ സൂക്ഷിക്കുന്നതാണ് നല്ലത്. പകരം, നിങ്ങൾ ചെയ്യുന്ന ഓരോ ആപ്ലിക്കേഷനും പ്രത്യേകം എൻവയോൺമെന്റുകൾ സൃഷ്ടിച്ച്, അവയ്ക്ക് ആവശ്യമായ പാക്കേജുകളും ഡിപെൻഡൻസികളും പ്രത്യേകം ഇൻസ്റ്റാൾ ചെയ്ത് ഉപയോഗിക്കാം. ഇതുവഴി, ഓരോ ആപ്ലിക്കേഷനും അതിന്റേതായ ഡിപെൻഡൻസികളും പാക്കേജുകളും ഉണ്ടായിരിക്കും. ഇതിലൂടെ മറ്റ് ആപ്ലിക്കേഷനുകളുമായി വരാൻ സാധ്യതയുള്ള പൊരുത്തക്കേടുകളെക്കുറിച്ചോർത്ത് നിങ്ങൾക്ക് ആശങ്കപ്പെടേണ്ടി വരില്ല.

പൈത്തണിൽ ഇത് സാധ്യമാക്കുന്നത് [*വിർച്വൽ എൻവയോൺമെന്റുകൾ (virtual environments)*](https://docs.python.org/3/tutorial/venv.html) വഴിയാണ്. ഓരോ ആപ്ലിക്കേഷനും ആവശ്യമായ എല്ലാ പാക്കേജുകളും, പൈത്തണിന്റെ ഒരു പ്രത്യേക പതിപ്പും അടങ്ങുന്ന ഒരു സ്വതന്ത്ര ഡയറക്ടറി ട്രീ (self-contained directory tree) ആണ് ഇവയോരോന്നും. ഇത്തരമൊരു വിർച്വൽ എൻവയോൺമെന്റ് നിർമ്മിക്കാൻ വ്യത്യസ്തങ്ങളായ ധാരാളം ടൂളുകൾ ലഭ്യമാണെങ്കിലും, ഇവിടെ നമ്മൾ പൈത്തണിന്റെ ഔദ്യോഗിക പാക്കേജായ [`venv`](https://docs.python.org/3/library/venv.html#module-venv) ആണ് ഉപയോഗിക്കുന്നത്.

ആദ്യം നിങ്ങളുടെ ആപ്ലിക്കേഷന് ആവശ്യമായ ഡയറക്ടറി നിർമ്മിക്കുക. ഉദാഹരണത്തിന്, നിങ്ങൾക്ക് ഇപ്പോൾ ഹോം ഡയറക്ടറിയിൽ നേരിട്ട് transformers-course എന്ന പുതിയ ഡയറക്ടറി ഉണ്ടാക്കണമെന്നിരിക്കട്ടെ:

```
mkdir ~/transformers-course
cd ~/transformers-course
```

ഈ ഡയറക്ടറിയുടെ ഉള്ളിൽ നിന്നുകൊണ്ടുതന്നെ, പൈത്തണിന്റെ `venv` മോഡ്യൂൾ ഉപയോഗിച്ച് ഒരു വിർച്വൽ എൻവയോൺമെന്റ് സൃഷ്ടിക്കാം:

```
python -m venv .env
```

ശൂന്യമായിരുന്ന ഈ ഫോൾഡറിൽ ഇപ്പോൾ നിങ്ങൾക്ക് *.env* എന്ന് പേരുള്ള പുതിയൊരു ഡയറക്ടറി കാണാനാകും:

```
ls -a
```

```out
.      ..    .env
```

നിങ്ങൾക്ക് `activate`, `deactivate` എന്നീ സ്ക്രിപ്റ്റുകൾ ഉപയോഗിച്ച് ഈ വിർച്വൽ എൻവയോൺമെന്റിലേക്ക് പ്രവേശിക്കാനും അതിൽ നിന്ന് പുറത്തുകടക്കാനും സാധിക്കും:

```
# Activate the virtual environment
source .env/bin/activate

# Deactivate the virtual environment
deactivate
```

ഇപ്പോൾ പറഞ്ഞ വിർച്വൽ എൻവയോൺമെന്റ് പ്രവർത്തനക്ഷമമായോ എന്ന് ഉറപ്പുവരുത്താൻ `which python` എന്ന കമാൻഡ് ഉപയോഗിക്കുക. ഇതിന്റെ ഔട്ട്പുട്ട് വിർച്വൽ എൻവയോൺമെന്റിന്റെ പാത്ത് (path) ആണ് കാണിക്കുന്നതെങ്കിൽ, നിങ്ങൾ അത് വിജയകരമായി ആക്ടിവേറ്റ് ചെയ്തിരിക്കുന്നു!

```
which python
```

```out
/home/<user>/transformers-course/.env/bin/python
```

### ഡിപെൻഡൻസികൾ (Dependencies) ഇൻസ്റ്റാൾ ചെയ്യൽ[[installing-dependencies]]

മുൻപത്തെ ഗൂഗിൾ കൊളാബ് വിഭാഗത്തിലെപ്പോലെതന്നെ, ഇവിടെയും ആവശ്യമായ പാക്കേജുകൾ ഇൻസ്റ്റാൾ ചെയ്യേണ്ടതുണ്ട്. മുൻപ് ചെയ്തതുപോലെ, `pip` ഉപയോഗിച്ച് 🤗 Transformers-ന്റെ ഡെവലപ്‌മെന്റ് വേർഷൻ നിങ്ങൾക്ക് ഇൻസ്റ്റാൾ ചെയ്യാവുന്നതാണ്:


```
pip install "transformers[sentencepiece]"
```

നിങ്ങൾ ഇപ്പോൾ മുന്നോട്ട് പോകാൻ പൂർണ്ണസജ്ജരാണ്!
